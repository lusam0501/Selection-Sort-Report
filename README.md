## 氣泡排序法

### 簡述原理
氣泡排序法（Bubble Sort）是一種相鄰比較的排序方式。  
它會從陣列開頭開始，依序比較相鄰兩個元素，若前者大於後者就交換位置。  
每完成一輪比較後，最大的元素就會像泡泡一樣慢慢浮到陣列尾端，因此稱為氣泡排序。

### 演算法流程
1. 從第一個元素開始，依序比較相鄰兩個元素。
2. 如果左邊元素比右邊元素大，就交換。
3. 一輪結束後，最後一個元素會是目前最大值。
4. 重複進行下一輪，比較範圍逐步縮小。
5. 若某一輪完全沒有交換，表示陣列已經排序完成，可提早結束。

### 複雜度分析
- 最佳時間複雜度: `O(n)`
- 平均時間複雜度: `O(n^2)`
- 最差時間複雜度: `O(n^2)`
- 空間複雜度: `O(1)`

### 特性
- 穩定排序
- 實作簡單
- 效率較差，不適合大量資料

### 模擬實作觀察
在模擬頁面中，可以清楚看到每一次相鄰元素的比較與交換。  
當資料量增加時，氣泡排序的交換次數與比較次數都明顯增加，因此執行速度較慢。  
若輸入資料本身接近排序完成，因為可以提早停止，所以實際表現會稍微好一些。

## 選擇排序法

### 簡述原理
選擇排序法（Selection Sort）的概念是：  
每一輪都從尚未排序的區間中找出最小值，並將它放到目前應該放置的位置。

### 演算法流程
1. 在未排序區間中找出最小元素。
2. 將此最小元素與未排序區間的第一個元素交換。
3. 已排序區間往右擴大一格。
4. 持續重複，直到所有元素排序完成。

### 複雜度分析
- 最佳時間複雜度: `O(n^2)`
- 平均時間複雜度: `O(n^2)`
- 最差時間複雜度: `O(n^2)`
- 空間複雜度: `O(1)`

### 特性
- 不穩定排序
- 比較次數固定較多
- 交換次數通常少於氣泡排序

### 模擬實作觀察
在模擬過程中，可以觀察到每一輪都會掃描整個未排序區間來尋找最小值。  
雖然它的交換次數不多，但比較次數仍然很多，因此整體效率仍不高。  
選擇排序的優點是流程規律、容易理解，適合作為排序演算法入門。

## 插入排序法

### 簡述原理
插入排序法（Insertion Sort）就像整理手上的撲克牌。  
每次從未排序區間取出一個元素，插入到前面已排序區間中的正確位置。

### 演算法流程
1. 假設第一個元素已排序。
2. 從第二個元素開始，取出目前元素。
3. 與前方已排序元素逐一比較。
4. 若前方元素較大，就往右移動。
5. 找到正確位置後，將目前元素插入。

### 複雜度分析
- 最佳時間複雜度: `O(n)`
- 平均時間複雜度: `O(n^2)`
- 最差時間複雜度: `O(n^2)`
- 空間複雜度: `O(1)`

### 特性
- 穩定排序
- 對小量資料很有效
- 對接近排序完成的資料表現不錯

### 模擬實作觀察
從模擬中可以看到，插入排序不一定會像氣泡排序那樣反覆交換，  
而是透過元素右移後再插入，讓排序過程較有結構。  
當資料接近有序時，插入排序所需的比較與移動次數會明顯減少，因此效率相對較好。

## 合併排序法

### 簡述原理
合併排序法（Merge Sort）採用分而治之（Divide and Conquer）的思想。  
它會先不斷將陣列對半切分，直到每個子陣列只剩一個元素，再逐步將小陣列合併回去，最後得到完整排序結果。

### 演算法流程
1. 將陣列分成左右兩半。
2. 分別對左右子陣列進行遞迴排序。
3. 將兩個已排序子陣列合併成一個新的有序陣列。
4. 重複直到整個陣列完成排序。

### 複雜度分析
- 最佳時間複雜度: `O(n log n)`
- 平均時間複雜度: `O(n log n)`
- 最差時間複雜度: `O(n log n)`
- 空間複雜度: `O(n)`

### 特性
- 穩定排序
- 效率穩定
- 需要額外記憶體空間

### 模擬實作觀察
在模擬頁面中，可以觀察到陣列先不斷被切分，再逐步合併。  
與前面幾種簡單排序不同，合併排序的重點不在於相鄰交換，而在於遞迴分割與合併。  
雖然過程較複雜，但在資料量大時，效率明顯優於 `O(n^2)` 的排序方法。

## 快速排序法

### 簡述原理
快速排序法（Quick Sort）同樣屬於分而治之法。  
它會先選定一個 pivot（基準值），將陣列分成小於 pivot 與大於 pivot 的兩部分，再對左右兩邊遞迴排序。

### 演算法流程
1. 選擇一個 pivot。
2. 將所有比 pivot 小的元素放到左邊，大的放到右邊。
3. pivot 會落在正確排序位置。
4. 對左右兩側子陣列重複進行相同操作。
5. 直到所有子陣列長度小於等於 1。

### 複雜度分析
- 最佳時間複雜度: `O(n log n)`
- 平均時間複雜度: `O(n log n)`
- 最差時間複雜度: `O(n^2)`
- 空間複雜度: `O(log n)`（遞迴堆疊）

### 特性
- 不穩定排序
- 平均效能非常好
- 最壞情況會退化
- 實務上非常常見

### 模擬實作觀察
在模擬中可以看到 pivot 的選擇會影響排序效率。  
若 pivot 選得好，左右區間較平均，速度就很快；  
若 pivot 每次都落在極端位置，就會導致遞迴不平均，使效率下降。  
因此快速排序雖然平均表現非常好，但仍有最壞情況需要注意。

## 綜合比較

### 各排序法比較表

| 排序法 | 最佳時間 | 平均時間 | 最差時間 | 空間複雜度 | 穩定性 |
|---|---|---|---|---|---|
| 氣泡排序法 | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 |
| 選擇排序法 | O(n^2) | O(n^2) | O(n^2) | O(1) | 不穩定 |
| 插入排序法 | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 |
| 合併排序法 | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 |
| 快速排序法 | O(n log n) | O(n log n) | O(n^2) | O(log n) | 不穩定 |

### 綜合分析
從複雜度來看，氣泡排序、選擇排序與插入排序都屬於 `O(n^2)` 的基礎排序法，較適合教學與小量資料。  
其中插入排序在資料接近有序時表現較佳，氣泡排序則較直觀，選擇排序則交換次數較少。

合併排序與快速排序屬於效率較高的排序法。  
合併排序的優點是時間複雜度穩定，缺點是需要額外空間；  
快速排序則在平均情況下速度很快，常用於實務應用，但需注意 pivot 選擇造成的最壞情況。

若依照使用情境來看：
- 小量資料或教學用途：插入排序、氣泡排序、選擇排序
- 大量資料且要求穩定：合併排序
- 大量資料且追求平均速度：快速排序

## 學習心得

透過這次排序報告與模擬實作，我更清楚理解了不同排序演算法之間的差異。  
以前只知道它們的名稱與公式，但在實際做出互動式模擬頁面後，才真正看懂每一步比較、交換、位移與分割的過程。

我發現基礎排序法雖然效率不高，但對理解排序概念很有幫助。  
例如氣泡排序可以清楚看到元素交換、選擇排序能理解「找最小值」的策略、插入排序則像整理撲克牌一樣直觀。  
而合併排序與快速排序則讓我學到分而治之與遞迴的設計思想，也知道演算法效率不只是看流程長短，而是要從整體複雜度分析。

這次實作最大的收穫，是把抽象的演算法轉化成可以觀察的畫面。  
透過視覺化之後，我更容易分辨每種排序法的特色，也能從操作過程中比較它們的優缺點。  
未來若還有機會，我希望能繼續加入更多排序法，例如 Heap Sort，並進一步比較它們在不同資料規模下的實際表現。
