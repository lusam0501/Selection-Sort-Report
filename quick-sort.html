快速排序法 Quick Sort

互動式視覺化教學 — 分區 · 遞迴 · 複雜度

一、操作原理

快速排序法（Quick Sort）也是一種經典的「分而治之」排序法。 它會先從陣列中選出一個基準值（pivot），然後重新排列資料， 讓比 pivot 小的元素放左邊，比 pivot 大的元素放右邊。

當 pivot 被放到正確位置後，左邊與右邊就形成兩個子問題， 再分別對它們做同樣的分區操作，最後整個陣列就會完成排序。

Quick Sort 平均情況下速度很快，實務上非常常見； 但如果每次都選到很差的 pivot，最壞情況會退化成 O(n²)。

二、演算法偽代碼

function quickSort(arr, low, high):
  if low < high:
    pivotIndex = partition(arr, low, high)
    quickSort(arr, low, pivotIndex - 1)
    quickSort(arr, pivotIndex + 1, high)

function partition(arr, low, high):
  pivot = arr[high]
  i = low - 1

  for j = low to high - 1:
    if arr[j] < pivot:
      i++
      swap(arr[i], arr[j])

  swap(arr[i + 1], arr[high])
  return i + 1

三、時間 & 空間複雜度

最佳情況 (Best): O(n log n)
每次 pivot 都能把資料切成接近平均的兩半。
平均情況 (Average): O(n log n)
一般情況下分區效果不錯，效率很高。
最壞情況 (Worst): O(n²)
每次 pivot 都落在極端位置，例如已排序資料又固定選尾端元素。
空間複雜度: O(log n)
主要來自遞迴呼叫堆疊，最壞可能到 O(n)。
穩定性: 不穩定 ✗
交換元素時，可能改變相等元素的原始相對順序。
