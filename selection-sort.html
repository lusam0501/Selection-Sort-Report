選擇排序法 Selection Sort

互動式視覺化教學 — 原理 · 步驟 · 複雜度

一、操作原理

選擇排序法（Selection Sort）的核心概念是： 每一輪從「尚未排序」的區間中找出最小值，然後把它放到前面正確的位置。

也就是說，第 1 輪找出整個陣列最小值放到第 0 格， 第 2 輪找出剩餘元素中的最小值放到第 1 格， 如此重複，直到整個陣列排序完成。

它的優點是概念簡單、交換次數通常不多；但缺點是即使資料幾乎排好， 仍然需要進行大量比較，因此效率通常不如更進階的排序法。

二、演算法偽代碼

function selectionSort(arr):
  n = arr.length
  for i = 0 to n - 2:
    minIndex = i
    for j = i + 1 to n - 1:
      if arr[j] < arr[minIndex]:
        minIndex = j
    if minIndex != i:
      swap(arr[i], arr[minIndex])

三、時間 & 空間複雜度

最佳情況 (Best): O(n²)
即使原本已排序，仍需逐輪掃描找最小值。
平均情況 (Average): O(n²)
每輪都要從剩餘區間中尋找最小元素。
最壞情況 (Worst): O(n²)
資料逆序或亂序時，仍需完整比較流程。
空間複雜度: O(1)
屬於原地排序，只需常數額外空間。
穩定性: 不穩定 ✗
交換最小值時，可能改變相等元素的原始相對順序。
