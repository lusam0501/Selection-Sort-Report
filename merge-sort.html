合併排序法 Merge Sort

互動式視覺化教學 — 分割 · 合併 · 複雜度

一、操作原理

合併排序法（Merge Sort）採用的是「分而治之」（Divide and Conquer）策略。 它會先把陣列不斷對半切開，直到每個子陣列只剩下一個元素； 因為單一元素本身就是已排序的。

接著再把這些小陣列兩兩合併，在合併過程中比較左右兩邊目前最前面的元素， 每次把較小者放進結果陣列，直到完成整段排序。

Merge Sort 的時間複雜度穩定，無論最佳、平均或最壞情況都為 O(n log n)， 因此是很經典也很重要的排序演算法。

二、演算法偽代碼

function mergeSort(arr, left, right):
  if left >= right:
    return

  mid = floor((left + right) / 2)

  mergeSort(arr, left, mid)
  mergeSort(arr, mid + 1, right)
  merge(arr, left, mid, right)

function merge(arr, left, mid, right):
  leftPart = arr[left...mid]
  rightPart = arr[mid+1...right]

  i = 0
  j = 0
  k = left

  while i < leftPart.length and j < rightPart.length:
    if leftPart[i] <= rightPart[j]:
      arr[k] = leftPart[i]
      i++
    else:
      arr[k] = rightPart[j]
      j++
    k++

  將剩餘元素依序放回 arr

三、時間 & 空間複雜度

最佳情況 (Best): O(n log n)
分割與合併流程固定，效率穩定。
平均情況 (Average): O(n log n)
每層處理 n 個元素，共約 log n 層。
最壞情況 (Worst): O(n log n)
即使資料分布不好，仍維持穩定效率。
空間複雜度: O(n)
需要額外陣列來儲存合併結果。
穩定性: 穩定 ✓
相等元素可以保留原本的相對順序。
